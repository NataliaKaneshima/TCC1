2. UMA PERSPECTIVA EVOLUTIVA SOBRE BIG DATA
    Esta seção apresenta uma visão geral das características do big data de um
perspectiva evolutiva.
    A McKinsey define big data como “os conjuntos de dados cujo tamanho está além
a capacidade de ferramentas típicas de software de banco de dados para capturar, 
armazenar, gerenciar e analisar ”[9]. 
    O Gartner define big data como os “ativos de informação de alto volume, alta velocidade 
e alta variedade que exigem formas de informação inovadoras e econômicas processamento 
para melhor percepção e tomada de decisão ”[5]. 
    Esta definição, por causa de seu recurso de marketing, tem sido usada com mais frequência 
nas comunidades de analistas e na academia comunidades de pesquisa e desenvolvimento de 
big data [1].

    De uma perspectiva evolutiva, Doug Laney usa 3 Ds: volume de dados, velocidade de dados e 
variedade de dados para representam as características dos dados no e-commerce em 2001 [10].
    Mais tarde, esses 3 Ds foram alterados para 3 Vs (volume, velocidade, e variedade) que 
foram explicadas como três características de big data [11, 12]. Além disso, esses 3 Vs 
foram estendidos primeiro a 4 Vs como quatro características de big data, adicionando 
veracidade [13, 14], e então para 5 Vs como cinco características de big data
(volume, variedade, velocidade, veracidade, valor) [15, 16], finalmente para 10 Vs adicionando 
outros 5 Vs: validade, variabilidade, local, vocabulário e imprecisão [17].
